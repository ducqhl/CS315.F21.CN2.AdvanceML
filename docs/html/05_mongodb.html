MongoDB — Serving Layer Single serving store cho toàn bộ hệ thống · 7+ collections

Design principle: MongoDB là single source of truth cho serving layer. Tất cả writers (Spark Streaming, Spark Batch, Inference Scheduler, Producer) ghi vào MongoDB. Tất cả readers (FastAPI, Streamlit) đọc từ MongoDB. Không có cache layer trung gian — giảm complexity vận hành.

Upsert Pattern cho Predictions

Inference Scheduler chạy mỗi 5 phút — idempotency là bắt buộc để tránh tích lũy duplicate documents.

# Key = (coin, prediction_date) — unique per coin per day
collection.update_one(
    filter={"coin": coin, "prediction_date": pred_date},
    update={"$set": document},
    upsert=True   # Insert if not exists, update if exists
)

# Chạy 2 lần → vẫn chỉ 7 documents (không phải 14)
# Sau 1 tháng (8640 lần) → vẫn chỉ 7 documents
Tại sao upsert idempotency quan trọng? Inference Scheduler chạy mỗi 5 phút = 288 lần/ngày = 8.640 lần/tháng. Nếu dùng insert thay vì upsert, sau 1 tháng sẽ có 60.480 duplicate prediction documents cho một coin. Upsert key (coin, prediction_date) đảm bảo mỗi cặp coin+ngày chỉ có tối đa 1 prediction.

TTL Index — Automatic Data Expiry

// realtime_prices: xóa sau 7 ngày
db.realtime_prices.createIndex(
    { "created_at": 1 },
    { "expireAfterSeconds": 604800 }  // 7 × 24 × 3600
)

// alerts: xóa sau 30 ngày
db.alerts.createIndex(
    { "timestamp": 1 },
    { "expireAfterSeconds": 2592000 }  // 30 × 24 × 3600
)
TTL cho realtime_prices: Với poll interval 600s, 1 tuần = 1.008 price ticks × 2 coins = 2.016 documents. TTL giữ collection nhỏ và queries nhanh. daily_stats không cần TTL vì là historical data dùng cho LSTM — cần giữ lại vĩnh viễn.

MongoDB Connection Configuration

MONGO_URI = "mongodb://admin:password123@mongodb:27017/crypto_db?authSource=admin"
# Trong Docker network: hostname = "mongodb" (service name trong docker-compose)
# Ngoài Docker (local dev): hostname = "localhost"

# Python connection via PyMongo
from pymongo import MongoClient
client = MongoClient(MONGO_URI)
db = client["crypto_db"]
Security: Credentials hardcoded trong docker-compose.yml và .env là cho demo. Production cần secrets management: Docker Secrets, HashiCorp Vault, hoặc Kubernetes Secrets. Không commit credentials vào Git — dùng .env.example làm template.

Trade-off: MongoDB vs. Alternatives

OptionPhù hợp khiHạn chế với hệ thống này
MongoDB (chọn) Flexible schema, throughput thấp (poll 600s), cần single serving store đơn giản Không tối ưu cho time series queries với hàng triệu records/ngày
PostgreSQL Cần ACID transactions, complex joins, fixed schema Schema migration phức tạp khi thêm collection mới. Cần ALTER TABLE.
InfluxDB Time series chuyên dụng, tick-by-tick data (hàng triệu/ngày) Overkill cho throughput thấp. ML pipeline cần flexible document format cho predictions.
Cassandra Horizontal scale, high write throughput, multi-datacenter Operational complexity quá cao cho single-machine demo. Query patterns đơn giản không cần Cassandra.
