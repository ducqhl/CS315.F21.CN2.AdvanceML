E2E Testing Strategy tests/ — 3 layer E2E tests + 7 unit test suites · testcontainers

Nguyên tắc thiết kế: E2E tests dùng infrastructure thật (Kafka container, MongoDB container qua testcontainers). Chỉ mock CoinGecko API để kiểm soát input và tránh phụ thuộc network ngoài. Unit tests dùng mock/stub hoàn toàn và không cần infrastructure.

Test Pyramid

Unit Tests — 7 suites

Không cần Docker infrastructure. Chạy ngay với make test.

  • test_lstm.py — model forward pass, loss, output shapes
  • test_indicators.py — SMA, RSI, VWAP, Bollinger correctness
  • test_batch_job.py — aggregation logic
  • test_producer.py — schema building, OHLC logic
  • test_mongo_writer.py — upsert pattern
  • test_dashboard.py — widget render, data loading
  • test_accuracy_tracker.py — comparison logic, metric calculation

E2E Tests — 3 suites

Cần Docker. Đánh dấu @pytest.mark.e2e. Chạy với make e2e. Loại khỏi default pytest run.

  • test_producer_kafka.py — Layer 1
  • test_batch_mongo.py — Layer 2
  • test_ml_mongo.py — Layer 3

Layer 1: Producer → Kafka

File: tests/e2e/test_producer_kafka.py

Xác nhận CoinGecko Producer hoạt động đúng với Kafka broker thật (testcontainers). Kafka container được spin up, Producer chạy 1 poll cycle, Consumer đọc và verify messages.

Test CaseAssertionTại sao quan trọng
test_produces_two_messages_per_cycle Đúng 2 messages: BTC + DOGE Contract: mỗi cycle đúng 2 coin. Thêm coin sẽ exceed CoinGecko quota.
test_message_schema_is_complete 12 required fields có đủ Spark Streaming CRYPTO_SCHEMA cần đủ fields — thiếu 1 field → parse error downstream.
test_only_btc_and_doge_produced Không có coin nào khác trong topic Scope isolation — ETH hoặc coin khác không được phép vào pipeline này.
test_ohlc_fields_populated_from_candles open/high/low/close từ OHLC data OHLC fetch logic riêng biệt — cần test độc lập với price fetch.
test_message_values_are_correct Giá trị khớp với mock CoinGecko data End-to-end correctness: không có data corruption qua JSON serialization.
Tại sao test này quan trọng? Đây là điểm đầu vào duy nhất của dữ liệu. Schema contract giữa Producer và Spark Streaming Consumer phải được giữ — nếu Producer thay đổi field names, toàn bộ downstream pipeline thất bại. Test này là safety net chống schema regression.

Layer 3: ML Pipeline → MongoDB

File: tests/e2e/test_ml_mongo.py

Xác nhận toàn bộ ML pipeline: từ LSTM training, lưu artifacts, đến inference và ghi predictions vào MongoDB.

ClassTestAssertion
TestLstmTraining test_training_returns_metrics Training pipeline trả về dict chứa RMSE, MAE, dir accuracy
test_model_file_saved lstm_{coin}_v2.pt tồn tại tại đúng path
test_scaler_file_saved scaler_{coin}.pkl tồn tại — cần cho inference inverse transform
test_rmse_is_non_negative RMSE ≥ 0
test_mae_is_non_negative MAE ≥ 0
test_directional_accuracy_in_range Dir accuracy ∈ [0, 1]
TestInferenceToMongo test_writes_7_predictions Inference ghi đúng 7 documents (horizon = 7)
test_prediction_document_schema Fields bắt buộc: coin, date, predicted_price, model_version
test_predicted_prices_are_positive Mọi giá dự đoán > 0
test_prediction_dates_are_in_future 7 ngày dự đoán đều sau ngày hiện tại
test_prediction_dates_are_unique Không có ngày trùng — mỗi ngày chỉ 1 prediction
test_upsert_idempotency Chạy inference 2 lần → vẫn chỉ 7 documents (không phải 14)
test_upsert_idempotency là test quan trọng nhất của Layer 3: Inference Scheduler chạy mỗi 5 phút = 288 lần/ngày = 8.640 lần/tháng. Nếu không idempotent → sau 1 tháng có 60.480 duplicate documents. Hệ thống có thể restart bất kỳ lúc nào mà không gây data corruption — đây là property thiết yếu cho production stability.

Chạy Tests

# Unit tests (không cần Docker)
make test
pytest tests/ -v --ignore=tests/e2e

# Specific test suites
make test-producer      # test_producer.py only
make test-lstm          # test_lstm.py only
make test-batch         # test_batch_job.py only
make test-dashboard     # test_dashboard.py + test_indicators.py + test_mongo_writer.py

# E2E tests (cần Docker running)
make e2e
make e2e-layer-1        # Producer → Kafka only
make e2e-layer-2        # Spark Batch → MongoDB only
make e2e-layer-3        # ML Pipeline → MongoDB only

# Một test cụ thể
pytest tests/test_lstm.py::TestLSTMModel::test_forward -v

# Với coverage report
pytest tests/ --cov=src --cov-report=html
# → htmlcov/index.html

pytest.ini Configuration

[pytest]
markers =
    e2e: marks tests as end-to-end (deselect with '-m not e2e')

# E2E tests excluded from default run
addopts = -m "not e2e"

testpaths = tests/
python_files = test_*.py
python_classes = Test*
python_functions = test_*
Design rationale: E2E tests dùng real Kafka/MongoDB containers (testcontainers) — bởi vì mock không đủ. Mock Kafka chỉ verify "có gọi producer.send()" — không verify message thực sự được delivered và consumer có thể đọc lại. Schema mismatch và serialization bugs chỉ xuất hiện với real infrastructure.
