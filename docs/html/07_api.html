FastAPI Backend — Serving Layer src/api/main.py · port 8000 · JWT stateless authentication

Trách nhiệm

FastAPI là lớp giao tiếp duy nhất giữa MongoDB và các client (React Frontend, Streamlit). Nó xử lý authentication, truy vấn MongoDB, merge batch + speed views, và format JSON response cho từng endpoint.

Sequence diagram request điển hình: React client → JWT middleware (verify token) → MongoDB query → JSON response. Không có caching layer — mỗi request là fresh MongoDB query. Acceptable vì throughput thấp (poll 600s).

JWT Authentication Flow

Client gửi POST /auth/login

FastAPI verify credentials, tạo JWT token với payload: {sub: user_id, exp: now + 24h}, ký bằng SECRET_KEY (HS256 algorithm).

Client lưu token ở localStorage

React Frontend lưu token. Axios interceptor tự động thêm Authorization: Bearer {token} vào mọi request tiếp theo. Client không cần quản lý manually.

FastAPI verify JWT mỗi request

Middleware decode token, verify signature với SECRET_KEY, kiểm tra expiry. 401 Unauthorized nếu token invalid, expired, hoặc missing.

Auto-refresh khi nhận 401

Axios response interceptor: nhận 401 → gọi POST /auth/refresh → lưu token mới vào localStorage → retry original request tự động. User không thấy interruption.

JWT vs Session Authentication

JWT (dùng trong đề tài)Session-based
Stateless — server không lưu gì. Token là self-contained. Stateful — server lưu session trong DB/Redis.
Scale horizontally dễ — mọi instance verify cùng SECRET_KEY. Scale phức tạp — cần shared session store (Redis) giữa instances.
Cannot revoke ngay lập tức (phải đợi expire). TTL 24h là trade-off. Có thể delete session ngay (logout immediate).
Phù hợp cho SPA + REST API. Token ở client. Phù hợp cho server-side rendered apps. Session ở server.

Merging Batch + Speed Views

Một số endpoints merge dữ liệu từ cả batch layer và speed layer để cung cấp view đầy đủ nhất:

@app.get("/api/prices/merged")
async def get_merged_prices(coin: str, days: int = 30, user = Depends(verify_jwt)):
    """
    Merge historical (batch) + realtime (speed) views.
    - Batch: daily_stats cho ngày trước (chính xác, đầy đủ)
    - Speed: realtime_prices cho hôm nay (approximate, low latency)
    """
    # Fetch historical from batch layer
    historical = db.daily_stats.find(
        {"coin_name": coin,
         "date": {"$gte": (datetime.now() - timedelta(days=days)).date().isoformat()}},
        sort=[("date", 1)]
    )

    # Fetch latest from speed layer
    realtime = db.realtime_prices.find_one(
        {"coin": coin.upper()},
        sort=[("event_time", -1)]
    )

    return {
        "historical": list(historical),
        "realtime":   realtime,
        "merged_at":  datetime.utcnow().isoformat()
    }

Inference Scheduler

Inference Scheduler chạy như background service độc lập, không phải qua API. Thực hiện hai nhiệm vụ theo chu kỳ 5 phút:

Task 1 — Run Inference

Load LSTM model từ disk, đọc 60+30 ngày gần nhất từ MongoDB/CSV, tính features, forward pass, upsert 7 predictions vào predictions collection.

Chu kỳ: Mỗi 5 phút

Task 2 — Track Accuracy

So sánh các predictions cũ với giá thực tế hiện tại. Tính directional accuracy rolling. Dữ liệu hiển thị trên trang Predictions của React Frontend.

Chu kỳ: Mỗi 5 phút (sau inference)
